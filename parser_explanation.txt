PARSER.PY - EXPLANATION
=======================

OVERVIEW
--------
This module is responsible for parsing PDF resumes and evaluating their quality based on two key metrics:
1. ATS (Applicant Tracking System) compatibility score
2. Design quality score

The module extracts text from PDF files and analyzes it to provide actionable feedback on resume quality.


DEPENDENCIES
------------
- pdfplumber: For extracting text and annotations from PDF files
- re: For regular expression pattern matching
- typing.Dict: For type hints


MAIN FUNCTIONS
--------------

1. extract_PdfText(pdf_path: str) -> str
   Purpose: Extracts all text content from a PDF resume, including hyperlinks
   
   How it works:
   - Opens the PDF file using pdfplumber
   - Iterates through each page and extracts text content
   - Handles cases where pages might not have extractable text
   - Extracts URI links from PDF annotations (clickable links)
   - Appends all links at the end of the extracted text
   - Returns the complete text as a single string
   
   Error handling:
   - Catches and prints any exceptions during PDF reading
   - Returns empty string if extraction fails


2. heuristic_atsScore(text: str) -> int
   Purpose: Calculates an ATS compatibility score (0-100) based on text quality
   
   Scoring logic (starts at 100, deducts points for issues):
   
   a) Empty/minimal content check:
      - Returns base score of 40 if text is empty
   
   b) Non-ASCII character ratio:
      - Counts ASCII printable characters (32-126) plus newlines/tabs
      - Calculates ratio of non-ASCII characters
      - Deducts up to 30 points if non-ASCII ratio > 2%
      - Rationale: ATS systems may struggle with special characters
   
   c) Excessive spacing:
      - Searches for 6+ consecutive spaces
      - Deducts 15 points if found
      - Rationale: Indicates formatting issues or table-based layouts
   
   d) Content length:
      - Deducts 20 points if total characters < 800
      - Rationale: Resume is too short to be meaningful
   
   Final score: Clamped between 40 and 100


3. design_score(text: str) -> int
   Purpose: Evaluates the visual design and formatting quality of the resume
   
   Scoring logic (starts at 70, adds/deducts points):
   
   a) Bullet points (POSITIVE):
      - Adds 10 points if bullet points (•, -, *) are detected
      - Rationale: Bullet points improve readability
   
   b) Long URLs (NEGATIVE):
      - Deducts 10 points if URLs longer than 60 characters are found
      - Rationale: Long URLs clutter the resume and reduce readability
   
   c) Excessive blank lines (NEGATIVE):
      - Deducts 5 points if 3+ consecutive newlines are found
      - Rationale: Too much whitespace wastes space
   
   d) Long lines (NEGATIVE):
      - Deducts 10 points if any line exceeds 140 characters
      - Rationale: Lines too long may indicate poor formatting
   
   Final score: Clamped between 40 and 100


4. parse_resume(filepath: str) -> Dict
   Purpose: Main entry point that orchestrates the complete resume parsing process
   
   Process:
   1. Extracts text from the PDF using extract_PdfText()
   2. Calculates ATS compatibility score using heuristic_atsScore()
   3. Calculates design quality score using design_score()
   4. Returns a dictionary with three keys:
      - "raw_text": The extracted text converted to lowercase
      - "ats_parse_rate": ATS compatibility score (40-100)
      - "design_score": Design quality score (40-100)
   
   Return format:
   {
       "raw_text": "lowercase text content...",
       "ats_parse_rate": 85,
       "design_score": 75
   }


SCORING SUMMARY
---------------

ATS Parse Rate (40-100):
- Measures how well ATS systems can parse the resume
- Higher score = better ATS compatibility
- Key factors: ASCII compatibility, spacing, content length

Design Score (40-100):
- Measures visual formatting and readability
- Higher score = better design quality
- Key factors: bullet points, URL length, whitespace, line length


USE CASE
--------
This module is typically used in a resume review application where:
1. User uploads a PDF resume
2. System calls parse_resume(filepath) to analyze it
3. Scores are displayed to the user with improvement suggestions
4. Raw text can be used for further analysis (keyword matching, etc.)


LIMITATIONS
-----------
- Only works with PDF files (not DOCX, TXT, etc.)
- Heuristic scoring may not perfectly match all ATS systems
- Design score is based on text patterns, not visual layout
- Converts all text to lowercase, losing case information
- Minimum score is 40, even for very poor resumes
